# Profile Page

## 개요

사용자 정보, 통계, 설정, 로그아웃 기능을 제공하는 프로필 페이지

---

## 경로

`/profile`

---

## 페이지 구조

```
┌────────────────────────────────────────────┐
│ 사용자 이메일                               │
├────────────────────────────────────────────┤
│ [총 북마크]  [폴더 수]  [즐겨찾기]          │
├────────────────────────────────────────────┤
│ 야간 모드              [토글 스위치]        │
├────────────────────────────────────────────┤
│ [로그아웃]                                  │
└────────────────────────────────────────────┘
│ [Bottom Navigation - 프로필 탭 활성]        │
└────────────────────────────────────────────┘
```

---

## 섹션별 상세

### User Info
- `UserInfo` 컴포넌트
- 사용자 이메일 표시

### Statistics
- `Statistics` 컴포넌트
- 3열 그리드로 통계 표시:
  - 총 북마크 수 (`total_bookmarks`)
  - 폴더 수 (`folder_count`)
  - 즐겨찾기 수 (`favorite_count`)

### Settings
- `SettingsList` + `SettingsItem` 컴포넌트
- **야간 모드**: `ThemeToggle` 컴포넌트 (next-themes 기반)
- 구분선 (`border-t border-border`) 위에 배치

### Actions
- `LogoutButton` 컴포넌트
- 로그아웃 API 호출 후 로그인 페이지로 이동

---

## 데이터 로드

- `useGetProfile` 훅으로 통합 API 호출
- API: `GET /api/auth/user` (이메일 + 통계 통합 응답)
- 프로필 데이터 없으면 `null` 반환 (아무것도 렌더링하지 않음)

---

## 기술적 특징

- SSR 비활성화 (`dynamic`)
- `useGetProfile` 단일 훅으로 모든 데이터 로드
